# Risk Scoring Engine

**Composite v2** student risk scoring for a dissertation prototype. Everything lives in **`risk_scoring_engine.py`**: the mathematical model at the top, then validation and CLI. The data generator imports the same functions from this file. Multi-metric stresses, contextual adjustments, intervention-history blend, then validation against `../data/students_data.csv`.

**Important:** Demonstration on **simulated data only** (no real students).

## Data location

| File | Location |
|------|----------|
| Student records | `../data/students_data.csv` |

New columns in the student file include: `accessed_upcoming_assessment_brief`, `days_to_nearest_assessment`, `risk_score_pre_intervention`, `prior_intervention_count`, `prior_interventions_improved`, `intervention_history_stress`.

## Model summary

See **`RISK_MODEL_SPEC.md`** and the docstring at the top of **`risk_scoring_engine.py`**. In short:

- Eight component stresses (attendance, VLE logins, VLE hours, grade, submissions, trajectory, GPA gap, assessment prep).
- Weighted composite → calibrated pre-score.
- Blend with **intervention history stress** from prior contacts and outcomes.
- Risk bands: Low 0–33, Medium 34–66, High 67–100.

## Installation

```bash
pip install -r requirements.txt
```

## Running

```bash
python risk_scoring_engine.py
```

1. Validate against the dataset (expect **100% exact** match if CSV was generated with the same generator and seed).
2. Interactive scoring (prompts for extended fields; optional fields use defaults).
3. Write `validation_report.txt`.
4. Exit.

## Programmatic use

```python
from risk_scoring_engine import calculate_risk_score, validate_against_dataset

out = calculate_risk_score({
    "attendance_rate": 65,
    "vle_logins_last_week": 3,
    "last_assessment_grade": 55,
    "assignments_submitted_late": 2,
})
print(out["risk_score"], out["risk_level"], out["risk_score_pre_intervention"])

validate_against_dataset("../data/students_data.csv")
```
